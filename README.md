# Auto-Drive

OpenCode TUI 插件 — AI 回复结束后自动继续下一轮。像自动驾驶一样驱动项目持续演进。

## 安装

```bash
# 从本地目录安装
opencode plugin add /path/to/auto-drive

# 或复制到项目 .opencode/plugins/
cp -r auto-drive .opencode/plugins/
```

启用后在 TUI 输入 `/auto-drive` 或按 `Ctrl+P` 搜索 `auto-drive` 打开模式菜单。

## 配置

配置文件按以下优先级合并（高优先级覆盖低）：

| 层级 | 路径 | 说明 |
|------|------|------|
| 全局 | `~/.config/opencode/auto-drive.json` | 用户级别默认配置 |
| 项目 | `.opencode/plugins/auto-drive.json` | 项目级别覆盖 |
| 插件选项 | `tui.json` 的 `pluginConfig` | 运行时最高优先级 |

### 配置格式

```json
{
  "mode": "ai",
  "maxTurns": 0,
  "customPrompt": "",
  "presets": {
    "继续优化": "继续优化当前功能，添加必要的注释和类型完善",
    "修复 Bug": "检查当前代码中的潜在问题并修复"
  }
}
```

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `mode` | string | `"stop"` | 当前模式: `stop` / `custom` / `ai` / 预设名称 / 序列名称 |
| `maxTurns` | number | `5` | 每 session 最大轮次，`0` = 无限 |
| `customPrompt` | string | `""` | 自定义模式的提示词 |
| `presets` | object | 内置 4 套 | 预设名称 → 提示词 |
| `sequences` | object | 内置 1 套 | 序列名称 → 提示词数组 |

## 模式

### ⏸ 停止
关闭自动驾驶，停止自动发送提示词。

### ✏️ 自定义
输入自定义提示词，每轮重复发送。适合需要固定指令的场景。

### 🤖 AI 驱动
发送引导 prompt，让 AI 自主分析项目并决定下一步方向。AI 会派出子代理从架构/代码质量/测试/文档/性能等视角并行分析，综合后决策执行。

### 🔄 序列
多提示词循环。定义一个提示词列表，每轮自动切换到下一个，循环执行。适合"分析→修复→测试→文档"等开发周期。

内置默认序列：

| 序列 | 步骤数 |
|------|--------|
| 🔄 完整开发周期 | 4 步（分析→修复→测试→文档） |

配置示例：
```json
{
  "sequences": {
    "完整开发周期": [
      "分析当前项目的代码结构和潜在问题",
      "修复发现的问题",
      "补充单元测试",
      "添加文档注释"
    ]
  }
}
```

状态栏显示：`🚀 AD 🔄 完整开发周期  2/∞  第2/4步`

### 📋 预设
内置 4 套预设，也可在配置中自定义：

| 预设 | 提示词 |
|------|--------|
| 📋 继续优化 | 继续优化当前功能，添加注释和类型完善 |
| 🐛 修复 Bug | 检查代码中的潜在问题并修复 |
| 🧪 补充测试 | 添加单元测试和集成测试 |
| 📝 添加文档 | 添加中文文档注释 |

## 使用

### 菜单
`/auto-drive` 或 `Ctrl+P` → `auto-drive` 打开模式菜单，选择模式后弹窗选择轮次。

### 快捷键
`Ctrl+Alt+A` — 快速切换启用/停止（恢复上次模式）。

### 轮次选择
选择模式后会弹窗选择执行轮次：无限 / 1 / 3 / 5 / 10 / 20 / 自定义。

### 状态栏
TUI 底部显示当前状态：

```
⏸ AD [继续优化]                 ← 停止（显示上次模式）
🚀 AD 🤖 AI 驱动  2/∞           ← AI 模式
🚀 AD 🐛 修复 Bug  3/5          ← 预设模式
🚀 AD "优化界面"  1/3           ← 自定义模式
🚀 AD 🔄 完整开发周期  2/5  第2/4步  ← 序列模式
```

每轮完成后 toast 通知：`🚀 第3轮完成 (3/5) | 修复 Bug`

## 完整配置示例

```json
{
  "mode": "完整开发周期",
  "maxTurns": 0,
  "customPrompt": "检查当前代码中的问题并修复，添加必要的注释",
  "presets": {
    "继续优化": "继续优化当前功能，添加必要的注释和类型完善",
    "修复 Bug": "检查当前代码中的潜在问题并修复",
    "补充测试": "为当前代码添加单元测试和集成测试",
    "添加文档": "为当前代码添加中文文档注释",
    "代码审查": "审查最近修改的代码，检查是否有 bug 或改进空间"
  },
  "sequences": {
    "完整开发周期": [
      "分析当前项目的代码结构和潜在问题",
      "修复第一步发现的问题",
      "为修改的代码补充单元测试",
      "为新增代码添加中文文档注释"
    ]
  }
}
```
