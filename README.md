# Auto-Drive

OpenCode TUI 插件 — AI 回复结束后自动继续下一轮。像自动驾驶一样驱动项目持续演进。

## 安装

```bash
# 从 npm 安装
opencode plugin add auto-drive

# 或从本地目录安装
opencode plugin add /path/to/auto-drive

# 或复制到项目 .opencode/plugins/
cp -r auto-drive .opencode/plugins/
```

启用后在 TUI 输入 `/auto-drive` 或按 `Ctrl+P` 搜索 `auto-drive` 打开模式菜单。输入 `/auto-drive-config`（或 `/adc`）查看当前合并的配置。

## 配置

配置文件按以下优先级合并（高优先级覆盖低）：

| 层级 | 路径 | 说明 |
|------|------|------|
| 全局 | `~/.config/opencode/auto-drive.json` | 用户级别默认配置 |
| 项目 | `.opencode/plugins/auto-drive.json` | 项目级别覆盖 |
| 插件选项 | `tui.json` 的 `pluginConfig` | 运行时最高优先级 |

### 配置格式

```json
{
  "mode": "ai",
  "maxTurns": 0,
  "customPrompt": "",
  "presets": {
    "继续优化": "继续优化当前功能，添加必要的注释和类型完善",
    "修复 Bug": "检查当前代码中的潜在问题并修复"
  }
}
```

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `mode` | string | `"stop"` | 当前模式: `stop` / `custom` / `ai` / 预设名称 / 序列名称 |
| `maxTurns` | number | `5` | 每 session 最大轮次，`0` = 无限 |
| `customPrompt` | string | `""` | 自定义模式的提示词 |
| `presets` | object | 内置 4 套 | 预设名称 → 提示词 |
| `sequences` | object | 内置 1 套 | 序列名称 → 提示词数组 |

## 模式

### ⏸ 停止
关闭自动驾驶，停止自动发送提示词。

### ✏️ 自定义
输入自定义提示词，每轮重复发送。适合需要固定指令的场景。

### 🤖 AI 驱动
发送引导 prompt，让 AI 自主分析项目并决定下一步方向。AI 会派出子智能体从架构/代码质量/测试/性能/错误处理/文档等视角并行分析，综合后决策执行。

### 🔄 序列
多提示词循环。定义一个提示词列表，每轮自动切换到下一个，循环执行。适合"分析→修复→测试→文档"等开发周期。

内置默认序列：

| 序列 | 步骤数 |
|------|--------|
| 🔄 完整开发周期 | 4 步（分析→修复→测试→文档） |

配置示例：
```json
{
  "sequences": {
    "完整开发周期": [
      "分析当前项目的代码结构和潜在问题",
      "修复发现的问题",
      "补充单元测试",
      "添加文档注释"
    ]
  }
}
```

状态栏显示：`🚀 AD 完整开发周期 第2/4步 2/5`

### 📋 预设
内置 4 套预设，也可在配置中自定义：

| 预设 | 提示词 |
|------|--------|
| 📋 继续优化 | 继续优化当前功能，添加注释和类型完善 |
| 🐛 修复 Bug | 检查代码中的潜在问题并修复 |
| 🧪 补充测试 | 添加单元测试和集成测试 |
| 📝 添加文档 | 添加中文文档注释 |

## 使用

### 菜单
`/auto-drive` 或 `Ctrl+P` → `auto-drive` 打开模式菜单，选择模式后弹窗选择轮次。菜单最底部有「📋 查看配置」入口。

### 快捷键
`Ctrl+Alt+A` — 快速切换启用/停止（恢复上次模式）。

### 轮次选择
选择模式后会弹窗选择执行轮次：无限 / 1 / 3 / 5 / 10 / 20 / 自定义。

### 状态栏
TUI 底部显示当前状态：

```
⏸ AD [继续优化]                 ← 停止（显示上次模式）
🚀 AD 🤖 AI 驱动  2/∞           ← AI 模式
🚀 AD 🐛 修复 Bug  3/5          ← 预设模式
🚀 AD 优化界面  1/3           ← 自定义模式
🚀 AD 完整开发周期 第2/4步 2/5  ← 序列模式
```

从第二轮开始每轮完成后 toast 通知：`🚀 第3轮完成 (3/5) | 修复 Bug`（第一轮无通知，避免启动时干扰）

## 完整配置示例

```json
{
  "mode": "完整开发周期",
  "maxTurns": 0,
  "customPrompt": "检查当前代码中的问题并修复，添加必要的注释",
  "presets": {
    "继续优化": "继续优化当前功能，添加必要的注释和类型完善",
    "修复 Bug": "检查当前代码中的潜在问题并修复",
    "补充测试": "为当前代码添加单元测试和集成测试",
    "添加文档": "为当前代码添加中文文档注释",
    "代码审查": "审查最近修改的代码，检查是否有 bug 或改进空间"
  },
  "sequences": {
    "完整开发周期": [
      "分析当前项目的代码结构和潜在问题",
      "修复第一步发现的问题",
      "为修改的代码补充单元测试",
      "为新增代码添加中文文档注释"
    ]
  }
}
```

## 日志与错误处理

插件的运行日志通过 `console.log` 输出，以 `[auto-drive]` 前缀标识：

| 日志 | 含义 |
|------|------|
| `[auto-drive] 🚀 N/M "prompt"` | 发送第 N 轮提示词 |
| `[auto-drive] ❌ error` | 发送失败（网络问题等） |
| `[auto-drive] 配置解析失败: path` | 配置文件 JSON 格式错误，该层级配置被忽略 |

### Toast 通知

| Toast | 触发条件 | 含义 |
|-------|----------|------|
| `Auto-Drive 已就绪 (...)` | 插件加载完成 | 插件已就绪，可输入 `/auto-drive` 使用 |
| `🚀 第N轮完成 (N/M) \| 模式` | 每轮对话完成（第一轮除外） | 当前轮次和模式 |
| `😵 自动驱动发送失败: 原因` | 提示词发送失败 | 网络错误或 API 异常 |
| `⚠️ maxTurns 配置无效，已回退到 5` | 配置文件 maxTurns 非数字/负数 | 配置错误，使用默认值 |
| `⚠️ 预设/序列 "名称" 与内置模式冲突，已忽略` | 预设/序列名称与 stop/custom/ai 重名 | 冲突条目被跳过 |
| `⚠️ 模式 "名称" 无有效提示词，自动驾驶不会触发` | 选择的模式没有有效提示词 | 提示用户检查配置 |
| `配置保存失败: 原因` | 写入配置文件失败 | 磁盘或权限问题 |

### 边界情况

| 场景 | 行为 |
|------|------|
| 配置文件 JSON 非法 | 该层级配置被静默忽略，回退到低优先级配置 |
| 模式设为不存在的序列名称 | 自动驾驶不生效（`isActive` 返回 false），无自动提示 |
| 配置保存失败 | toast 通知用户，错误输出到 console |
| 新 session 刚创建 | 自动重试最多 2 次（间隔 500ms、1000ms），等待 session 就绪 |
