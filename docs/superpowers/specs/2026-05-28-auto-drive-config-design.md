# Auto-Drive 配置与多模式设计

## 概述

Auto-Drive 插件让 AI 回复结束后自动继续下一轮。本次扩展增加三种模式支持（自定义、预设、AI 驱动），用户可通过 `/auto-drive` 菜单切换。

## 配置

### 层级

| 层级 | 路径 | 优先级 |
|------|------|--------|
| 全局用户配置 | `~/.config/opencode/auto-drive.json` | 低（被项目覆盖） |
| 项目级配置 | `.opencode/plugins/auto-drive.json` | 高 |

### 格式

```json
{
  "$schema": "auto-drive",
  "mode": "stop",
  "presets": {
    "继续优化": "继续优化当前功能，添加必要的注释和类型完善",
    "修复 Bug": "检查当前代码中的潜在问题并修复",
    "补充测试": "为当前代码添加单元测试和集成测试",
    "添加文档": "为当前代码添加中文文档注释"
  },
  "customPrompt": ""
}
```

### 读取策略

1. 先读项目级 `.opencode/plugins/auto-drive.json`（若有）
2. 再读全局 `~/.config/opencode/auto-drive.json`（若有）
3. 以项目级为准逐字段覆盖
4. 最终结果与插件 `options` 合并（插件级最高优先级）

## 模式

### 1. 停止 (`stop`)

关闭自动驾驶。底部状态栏显示 `⏸`。

### 2. 自定义 (`custom`)

用户手写一行提示词。每轮重复发送该提示词。

- 选中后弹出 DialogPrompt 输入框
- 保存到配置 `customPrompt`
- 状态栏：`🚀 AD "用户提示词"`

### 3. 预设 (`preset`)

用户从内置或自定义预设列表中选择。

- **内置默认预设**：
  - 继续优化 — *继续优化当前功能，添加必要的注释和类型完善*
  - 修复 Bug — *检查当前代码中的潜在问题并修复*
  - 补充测试 — *为当前代码添加单元测试和集成测试*
  - 添加文档 — *为当前代码添加中文文档注释*
- 用户可在配置 `presets` 中增删改
- 选中后该预设为当前模式
- 状态栏：`🚀 AD 🐛 修复 Bug`（左为模式图标，右为预设名）

### 4. AI 驱动 (`ai`)

不发送固定 prompt。`session.idle` 时，发送一条引导 prompt 让 AI 自主分析项目状态并决定下一步方向：

> "基于当前项目和对话上下文，决定下一步最有价值的事。"
> "可以改 bug、重构、加新功能、加文档——完全由你判断。先输出计划再执行。"

- 状态栏：`🚀 AD 🤖`

## 交互

### `/auto-drive` 菜单

```
用户输入 /auto-drive 或 Ctrl+P → auto-drive
      ↓
  菜单（DialogSelect）
    ├─ ⏸ 停止
    ├─ ✏️ 自定义
    ├─ 🤖 AI 驱动
    ├─ ──────────────
    ├─ 📋 继续优化     (预设)
    ├─ 🐛 修复 Bug     (预设)
    ├─ 🧪 补充测试     (预设)
    └─ 📝 添加文档     (预设)
      ↓
  DialogSelect 选中后立即生效
  若选择"自定义"，再弹 DialogPrompt 输入提示词
```

### 状态栏（`app_bottom` 插槽）

| 模式 | 显示 |
|------|------|
| stop | `⏸` |
| ai | `🚀 AD 🤖` |
| custom | `🚀 AD "继续优化"` |
| preset | `🚀 AD 🐛 修复 Bug` |

横向布局，紧凑显示，适合底部状态栏一行空间。

### 命令更新

原 `/auto-drive` 命令从"切换开关"改为"打开模式选择菜单"。删除旧的 toggle 行为。

## AI 驱动执行逻辑

```
session.idle 触发
  → 检查 mode === "ai"
  → 构建引导 prompt：
      "基于当前项目和对话上下文，决定下一步最有价值的事。
       可以改 bug、重构、加新功能、加文档——完全由你判断。
       先输出计划再执行。"
  → 发送到当前会话
```

## 文件影响

- `tui.jsx` — 主逻辑：配置加载、模式菜单、三种模式分发、状态栏
- `package.json` — 不变
- `.opencode/plugins/auto-drive.json` — 项目配置（可选）
- `~/.config/opencode/auto-drive.json` — 全局配置（可选）
